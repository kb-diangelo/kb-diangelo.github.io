Week 7

JavaScript Constructor Functions vs. Ruby Classes

12/07/2015

Last week we went pretty in depth about Ruby classes but as we are learning more and more about JavaScript, we're going to touch on something that's sort of an equivalent (but not really) to creating a class in Ruby: JavaScript constructor functions.

Recall: "In Ruby, a class is like a bundle of related methods (behaviors) and properties that are shared between different instances of that class (different instances of that object)." In JavaScript, constructor functions also lay out a blueprint for creating objects, but what the word "object" means in both languages is very different!

A Ruby object is anything. Like, anything. Everything is Ruby is considered an object and ultimately belongs to the Object class. A JavaScript object, on the other hand, is a particular kind of data structure, not completely dissimilar from a Ruby hash, in which value are stored under names (properties) of that object and then referred back to with dot notation. An example:

var maxolotl = {
species: "Ambystoma mexicanum",
name: "Maxolotl",
age: 5,
color: "leucistic",
cute: true,
bump: function(){console.log("Bump!")}
}

console.log(maxolotl.name);
console.log(maxolotl.age);
maxolotl.bump();

... this would print "Ambystoma mexicanum", 5, and "Bump!" to the console.

So now that we understand what an object is in JavaScript, how can we create lots of similar objects easily? The way to to do this is to create a function that constructs these objects given arguments about what the particular properties of the object are. Constructor functions look pretty similar to normal functions in JavaScript; there's no special notation that describes them as constructors. The only thing that is different from every other JavaScript function is the use of this.property = argument or value to set the properties of the objects to be constructed, and the convention that their names should be capitalized. So let's look at an Axolotl constructor.

function Axolotl(name, age, color) {
this.name = name;
this.age = age;
this.color = color;
// and some things that are true for all axolotls
this.cute = true;
this.species = "Ambystoma mexicanum";
this.bump = function(){console.log("Bump!")}
}

var max = new Axolotl("Maxolotl", 5, "leucistic");
var billy = new Axolotl("Billy Bob", 1, "wildtype");

Now we have two Axolotl objects that share certain traits (the bump function, being cute, being "Ambystoma mexicanum") while varying based on the input given (name, age, and color). Notice that we use the new keyword to create them.

So what are the key differences between Ruby classes and JavaScript constructor functions, besides the syntax? Well, like I said before, a Ruby class can define any kind of new object, but a JavaScript constructor can only create a new instance of the "object" data structure. There is also a matter of inheritance. Ruby classes can inherit from other classes, and be nested into one another, but constructor functions (AS I HAVE DESCRIBED THEM HERE) cannot (there may be other ways to inherit in JavaScript that I haven't learned about yet!). Also, in Ruby, if you want the ability to read/write to the properties of an instance of a class, you have to include that when you define the class (with attr_reader, attr_writer, and attr_accessor). In JavaScript, you can always(?) access the properties of an object built with a constructor using dot notation.
