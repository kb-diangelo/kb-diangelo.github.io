Recursion

12/13/2015

This week I'm going to give a very short introduction to what, if I remember correctly from the only computer science class I ever took in college, is a pretty central idea in computer science. Recursion is a process in which something is nested between larger and smaller versions of itself, like Russian dolls. In computer science, it refers to solving a problem by solving smaller instances of that problem (which may themselves be solved by solving even smaller instances of themselves and so on). Recursion is what happens when you have a function or a method call itself, making a loop.

Recursion would have been helpful in solving the factorial problem we had in Week 4. The solution (in Ruby) that I used looked like this:

def factorial(number)
if number == 0
return 1
else
total = 1
until number == 1
total = total * number
number = (number-1)
end
return total
end
end

As you can see, I use a while loop to solve this problem. But recursion offers us a better way. If I allow factorial to call itself...

def factorial(number)
if number == 0
return 1
else
return number * factorial(number-1)
end
end

For example if the number we input is 5, the return would be 5 * 4 * 3 * 2 * 1 * 1 = 120. Every time the number was not 0, factorial calls itself and finds a smaller number and multiplies it by the number that it had. Over and over. It's a bit tricky to wrap your head around but basically that's how it works.

Recursive functions/methods usually have at least one case like, in the example above, if number == 0, a case in which the function will not recurse, or only up until which it will recurse. This is called the base case. Other case are called the recursive cases. Forgetting to put in a base case can result in a function or method that calls itself forever in an infinite loop, which is clearly something to avoid!
