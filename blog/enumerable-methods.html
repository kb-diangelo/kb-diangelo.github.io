Week 5

Enumerable#cycle

11/23/2015

Today we're going to be talking about the enumerable method cycle. I'm going to keep this pretty brief but stick with me!

cycle is a method found in the enumurable module. This was my first experience with modules but they're a pretty cool idea. A module is kind of like a class in that it is a collection of behaviors: methods, constants, etc. In fact, Class is a subclass of Module. But instead of defining a particular object and its behaviors, modules allow you to share behaviors among certain types of objects, almost like a library of behaviors.

So what kinds of objects can use this library? Well, anything with an each method. So that means arrays, hashes, ranges, etc, as well as classes you create as long as you define a .each method for that class! (to make sure that the class has access to enumerable methods when you define the class you need to write include Enumerable when you define it).

So now that we know what the enumerable module is, what kind of methods are available in that "library" for us to use with our objects? Today I onl have time to discuss one: cycle. Cycle is fairly simple.

.cycle works very similarly to .each but instead of yielding the objects one at a time until the end of the data structure, it yields them over and over forever (or until a break). It takes an optional argument which is an integer number that tells you how many times to, well, cycle. You can also include a code block, just like with .each, that tells the method what to do with each item it picks out of the data structure.

Let's look at an example...

my_array = ["K", "B", "D"]
my_array.cycle(3) {|item| puts item}

=>
K
B
D
K
B
D
K
B
D

Here we cycled through my little array 3 times and puts each item to the console. If we didn't include the the (3) argument the cycle would continue forever.

Well that's the short version of cycle! I hope you enjoyed reading about what I learned this week at DBC!
