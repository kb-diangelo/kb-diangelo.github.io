Week 4

Arrays and Hashes

11/16/2015

This week at DBC, we learned about two really important Ruby data structures: arrays and hashes. These structures are similar in a few ways but there is a critical difference in how you access values in an array vs. in a hash.

Arrays are basically ordered lists of information. An array contains a collection of values - which can be any object: integers, strings, other arrays, booleans, etc - all of which have an index that represents their place in the array. These indices start at 0, not 1, as is typical in the computer world. So you can imagine an array as an ordered list of items, starting at 0.

You can create an array in a few ways. The first is to use the constructor Array.new...
my_array = Array.new()
... which will create a new, empty array. You can add items to the array either by pushing them to the array using << or .push, which will add them to the end of the array ...
my_array.push("first item")
my_array << "second item"
which will, for example, add the strings "first item" and "second item" to the empty array we created in index 0 and 1, respectively, or by specifying what index they should be at ...
my_array[2] = "third item"
my_array[3] = "fourth item"
which will, in this example, add the strings "third item" and "fourth item" to the array we created in index 2 and 3, respectively. We could have even done that in the opposite order, it would have been fine.

You can also create an array by using literal notation. You can initialize the array either as an empty array and then add items just like we did when we used the constructor, or add the values when you make the array, seperated by commas.

my_array = [] #creates an empty array
my_array << "first item"
my_array << "second item"
my_array << "third item"
my_array << "fourth item"
or
my_array = ["first item", "second item", "third item", "fourth item"]

To access the items in an array, we refer to them by their index. For example ...

my_variable = my_array[3]
would assign the value stored in the 3rd index of my_array (the string "fourth item") to my_variable.

To iterate over an array, that is, to do something for every value contained in the array, we can use the .each method. The syntax for this is a little more complicated than we've seen before. We need .each and a block bracketed by either do/end or the curly brackets {} to tell Ruby what we're doing for each of these items in the array. We also need a placeholder value bracketed by || (whatever that character is called), to be, as Chris Pine puts it, "the variable that each will use to point to the objects in the array" (Learn to Program 2nd Edition Ch. 8).

my_array.each do |value|
puts value
end
The code above, therefore, will iterate over all the values in my_array and puts each one to the console.

A hash is similar to an array in that it is a list of values but instead of these values having indices they have what are known as keys. A key is a value associated with the value stored in the hash. It can technically be any object, but the most common keys are strings and symbols.

Like an array, a hash can either be created with a constructor, in this case Hash.new() or literally.

# using a constructor
my_hash = Hash.new()
my_hash["favorite animal"] = "axolotl"
my_hash[:favorite_food] = "beets"

# using literal notation
# there are a couple ways to do this
# if your keys are symbols you can put the :
# at the end of the symbol instead of the front:
my_hash = {favorite_animal: "axolotl", favorite_food: "beets"}
# if your keys are strings, other values, symbols, or a mix,
# you can use the "hash rocket"
my_hash = {:favorite_animal => "axolotl", "favorite food" => "beets"}
# you can also add these values one at a time
# like we did in the above example

To iterate over a hash, you can use .each with two placeholder values in the ||, one for the key and one for the value. Alternatively, if you want to iterate over just keys or just values you can use the .each_key or .each_value methods respectively.

So why would you use one or the other? Arrays are good when you just need a list of items. Hashes are good when you want to associate a key with a value. To use the example of my favorite things, we could make a array of my favorites: "axolotl", "beets", and so on, but this array probably wouldn't make much sense. When we are storing information that needs some context, ie, a key, a hash is a better choice.
